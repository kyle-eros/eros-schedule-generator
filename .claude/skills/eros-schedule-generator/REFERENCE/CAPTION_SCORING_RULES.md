## Scoring Factor Weights

### Primary Scoring Factors

These weights determine caption selection priority after HARD FILTERS (vault matrix + AVOID tier exclusion) are applied.

| Factor | Points | Description | Calculation Method |
|--------|--------|-------------|-------------------|
| **Structural Pattern Match** | 0-30 | Presence of proven structural patterns | Alert emoji (+10), Duration (+10), Enumeration (+10) |
| **Language Power Signals** | 0-25 | High-impact language elements | Superlatives (+8), Climax (+8), Authenticity (+5), Intimacy (+4) |
| **Length Optimization** | 0-20 | Character count within optimal range | Within optimal (+20), Near optimal (+10), Outside (0 to -30) |
| **Temporal Freshness** | 0-15 | Days since last use by this creator | 60+ days (+15), 30-60d (+10), 14-30d (+5), <14d (0), <7d (-10) |
| **Diversity Contribution** | 0-10 | Uniqueness vs recent captions | New content type (+10), Structural variety (+5), Similar (-15) |
| **Persona Alignment** | 0-10 | Match with creator's persona | Strong match (+10), Compatible (+5), Mismatch (-10) |
| **Performance History** | 0-30 | Caption's historical earnings | Top 10% (+30), Top 25% (+20), Top 50% (+10), Below avg (-5) |
| **Send Type Compatibility** | 0-10 | Caption type matches send type | Perfect match (+10), Compatible (+5), Mismatch (-15) |

## Anti-Patterization Safeguards

These safeguards prevent pattern over-application and maintain creator uniqueness.

### Safeguard 1: Creator-Relative Baseline

**Purpose**: Patterns are evaluated relative to each creator's top performers, not universal benchmarks.

**Implementation**:
```python
# Extract baseline patterns from creator's top 10 earners
baseline_patterns = analyze_top_performers(
    creator_id=creator_id,
    top_n=10,
    lookback_days=90
)

# Score candidate captions against creator-specific baseline
pattern_match_score = compare_to_baseline(
    candidate_caption=caption,
    baseline_patterns=baseline_patterns
)
```

**Example**:
- Creator A's top earners: Heavy emoji use, playful tone, short captions
- Creator B's top earners: Minimal emojis, professional tone, long captions
- Same caption scored differently for each creator based on their baseline

**Result**: No universal "best caption" - only "best caption for Creator X"

---

### Safeguard 2: Temporal Decay Function

**Purpose**: Patterns lose effectiveness with repeated use (audience fatigue).

**Implementation**:
```python
# Pattern decay based on recent usage
decay_multiplier = 1.0 - (pattern_usage_count * 0.15)
decay_multiplier = max(decay_multiplier, 0.3)  # Floor at 30% effectiveness

adjusted_pattern_score = base_pattern_score * decay_multiplier
```

**Decay Schedule**:
| Usage Count (30d) | Decay Multiplier | Effective Score |
|-------------------|------------------|-----------------|
| 0 (new) | 1.0 | 100% |
| 1 | 0.85 | 85% |
| 2 | 0.70 | 70% |
| 3 | 0.55 | 55% |
| 4 | 0.40 | 40% |
| 5+ | 0.30 | 30% (floor) |

**Example**:
- Alert emoji (ðŸš¨) base score: +10 points
- Used 0 times in 30d: +10 points (full value)
- Used 2 times in 30d: +7 points (70% value)
- Used 5 times in 30d: +3 points (30% floor)

**Result**: High-performing patterns don't become overused templates

### Safeguard 3: Weekly Diversity Gate

**Purpose**: Prevent caption similarity within same week's schedule.

**Implementation**:
```python
# Check similarity to all captions already scheduled this week
for scheduled_caption in current_week_schedule:
    similarity = calculate_similarity(
        candidate_caption,
        scheduled_caption
    )

    if similarity > 0.60:  # HARD THRESHOLD
        reject_caption(reason="Exceeds weekly diversity threshold")
```

**Similarity Components**:
- Structural similarity (40%): Pattern overlap, format, length
- Language similarity (30%): Keyword overlap, tone, vocabulary
- Emoji similarity (20%): Emoji sequence and density
- Content type similarity (10%): Same content type referenced

**Threshold**:
- >0.60 similarity to ANY scheduled caption this week = REJECT
- Forces variety even if pattern scored well

**Example**:
- Monday PPV: "ðŸš¨ 43 minutes of HOTTEST B/G content ðŸ”¥"
- Wednesday PPV candidate: "ðŸš¨ 38 minutes of BEST B/G action ðŸ”¥"
- Similarity: 0.72 (REJECT - too similar to Monday)
- Alternative selected with lower similarity score

**Result**: Each week's schedule has visible variety for subscriber experience
